## 🔧 配置选项

### 必需配置

```go
// 时间范围分片
Config{
    ShardingType:        "time_range",
    TimeColumn:          "created_at",
    TimeRangeFormat:     "month",
    PrimaryKeyGenerator: sharding.PKSnowflake,
}

// 取模分片
Config{
    NumberOfShards:      10,
    ShardingKey:         "id",
    PrimaryKeyGenerator: sharding.PKSnowflake,
}
```

### 可选配置

- [x] `DoubleWrite` - 双写模式
- [x] `ShardingAlgorithm` - 自定义算法
- [x] `ShardingSuffixs` - 自定义后缀
- [x] `PrimaryKeyGeneratorFn` - 自定义主键

## ⚠️ 注意事项

### 使用建议

1. ✅ 在时间字段上建立索引
2. ✅ 定期预创建未来时间的表
3. ✅ 监控各分片表的数据分布
4. ✅ 设置合理的归档策略
5. ✅ 确保服务器时区正确

### 限制说明

1. ⚠️ 跨表查询性能较低
2. ⚠️ 需要合理选择分片粒度
3. ⚠️ 时间范围分片不适合非时序数据
4. ⚠️ 分片策略确定后不建议修改
